Study Notes

Hardware

  • Input devices: keyboard, mouse, scanner, microphone, webcam, touchscreen, biometric scanner.
  • Output devices: monitor, printer, speakers, projector.
  • Storage devices: HDD (magnetic, cheap, large capacity), SSD (faster, no moving parts, more expensive), flash drives, cloud storage, optical discs.
  • Processing: the CPU (Central Processing Unit) fetches, decodes, and executes instructions; speed measured in GHz.
  • RAM vs ROM: RAM is volatile, temporary working memory; ROM is non-volatile and stores permanent startup instructions.
🔑 Volatile memory: memory that loses its contents when power is switched off (e.g. RAM).

Software

  • System software: the operating system (e.g. Windows, macOS) and utility programs that manage hardware and support application software.
  • Application software: programs used to perform specific tasks — word processors, spreadsheets, browsers, games.
  • Open source vs proprietary software: open source is free with editable source code (e.g. Linux); proprietary is licensed and closed-source (e.g. MS Office).

Buying & Maintaining a Computer

  • Match hardware specifications (CPU speed, RAM, storage) to the intended use of the computer.
  • Keep software updated, run antivirus scans, defragment HDDs, and back up data regularly to maintain system health.

Data vs Information

🔑 Data: raw, unprocessed facts and figures. Information: data that has been processed to become meaningful and useful.

Database Concepts

  • Table: stores related data in rows (records) and columns (fields).
  • Primary key: a field that uniquely identifies each record in a table.
  • Foreign key: a field in one table that links to the primary key of another table, creating a relationship.
  • Queries: used to search, filter, and retrieve specific data from one or more tables.
  • Forms and reports: forms are used to capture/view data easily; reports present data in an organised, printable format.

Information Management Process

  1. Identify the problem/task.
  2. Find/collect relevant data (search techniques, evaluating sources).
  3. Process the data into information (spreadsheets, databases).
  4. Present the information appropriately (charts, reports, documents).
🔑 Data validity/reliability: checking that information found online is accurate, current, and from a credible, unbiased source.

Digital Citizenship & Ethics

  • Netiquette: acceptable, polite standards of behaviour when communicating online.
  • Digital divide: the gap between people who have easy access to computers/the internet and those who do not.
  • Green computing: using computing resources in an environmentally responsible way — e.g. energy-efficient hardware, recycling e-waste.

Threats & Protection

  • Malware: viruses, worms, trojans, spyware, and ransomware that damage or exploit systems.
  • Phishing: fraudulent attempts (usually via email) to trick users into revealing personal/financial information.
  • Firewall: software/hardware that monitors and controls incoming and outgoing network traffic.
  • Protection measures: antivirus software, strong passwords, two-factor authentication, regular updates, and backups.
🔑 Cyberbullying: using digital technology to deliberately and repeatedly harass, threaten, or embarrass another person.

Health & Ergonomics

  • Repetitive strain injury (RSI), eye strain, and back problems can result from poor posture and prolonged computer use.
  • Ergonomic practices: correct chair height, screen at eye level, regular breaks, wrist supports.

Network Basics

  • LAN (Local Area Network): connects devices within a small area, e.g. a school or office.
  • WAN (Wide Area Network): connects networks over large geographic distances — the Internet is the largest WAN.
  • Network hardware: routers direct data between networks; switches connect devices within a LAN; network cards (NICs) allow devices to connect.
  • Wi-Fi: wireless technology allowing devices to connect to a network without cables.
🔑 Bandwidth: the amount of data that can be transferred over a network connection in a given time, usually measured in Mbps.

Internet Services

  • ISP (Internet Service Provider): a company that provides access to the internet.
  • Cloud computing: storing and accessing data/programs over the internet instead of a local hard drive.
  • Email, VoIP, instant messaging, and video conferencing are common internet-based communication methods.
  • e-Commerce: buying and selling goods/services online.

Word Processing & Spreadsheets

  • Word processing skills: styles, mail merge, tables of contents, sections, headers/footers.
  • Spreadsheet skills: formulas, functions (SUM, IF, VLOOKUP, COUNTIF), absolute vs relative cell references, charts.
🔑 Absolute cell reference: a cell reference (e.g. $A$1) that does not change when a formula is copied to other cells.

Database & Presentation Software

  • Database skills: designing tables, setting primary keys, building queries, forms, and reports.
  • Presentation software: used to communicate information visually through slides — should be concise, well-designed, and not overcrowded with text.

Integration

  • Integration: combining data/objects from one application into another, e.g. inserting an Excel chart into a Word report or mail-merging a database into a document.
  • Problem-solving in Solution Development: analyse the task, plan the solution, choose the right application, implement, and check/evaluate the result.

Exam Paper Guide

Understand exactly what the Grade 11 CAT Theory paper looks like and how to maximise your marks.

📄 Theory Paper Overview

  • Duration: 2 hours
  • Total marks: 150
  • Section A: Short-answer/matching/multiple-choice questions
  • Section B & C: Systems Technologies, Networks, Data Management, Social Implications, Solution Development scenarios

🔑 Command Words

  • Name/List/State: 1 mark per correct fact, no explanation.
  • Explain/Describe: fact plus reasoning or example, usually 2 marks.
  • Give a reason/Motivate: justify a choice or opinion with logic.
  • Compare: identify similarities and/or differences between two concepts.

Mark Allocation Guide

Question typeMarks per pointWhat's expected
Name/List/State1 markA single correct fact or term.
Explain/Describe2 marksA fact plus a reason or short motivation.
Scenario-based application2 marksA concept applied specifically to the scenario given in the question.
📌 Exam tip: Use CAT-specific terminology exactly as taught — "RAM" not "memory," "malware" not "virus stuff." Vague wording loses marks.

Practical Exam Skills Guide

The Practical paper tests your ability to actually use Word, Excel, Access, and PowerPoint under exam conditions — here's how to approach it.

1

Read the Whole Question Paper First

Skim through all questions before starting so you know how much time to budget per application (Word, Excel, Access, PowerPoint) and don't get stuck on one section.

2

Save Files Correctly and Often

Always save with the exact file name given in the instructions, in the correct location, and in the correct file format (e.g. .docx, .xlsx). Save regularly to avoid losing work.

3

Follow Instructions Exactly

Markers check for exact instructions — correct cell references, exact formula syntax, exact formatting. "Close enough" answers often lose marks in CAT practical marking.

Example: If told to use a function to calculate total sales in cell D10, use =SUM(D2:D9) exactly — not a manually typed total.
4

Check Formulas and Cell References

Use absolute references ($A$1) where a formula must be copied without the reference changing, and relative references where it should change. Test formulas by copying them down/across before moving on.

Common Mistakes to Avoid

❌ Wrong File Name/Format

Saving as the wrong file type or with the wrong name means the marker can't find or open your work — an automatic loss of marks.

❌ Typing Values Instead of Formulas

Manually typing a calculated answer instead of using a formula/function loses the method marks the question was testing.

❌ Ignoring Exact Instructions

Using the wrong cell, wrong field name, or wrong formatting (even if the output "looks right") can cost marks.

❌ Running Out of Time on One Section

Don't get stuck perfecting one database query while leaving the presentation section untouched — attempt every section.

💻 Practice tip: Time yourself doing a past practical paper start to finish under exam conditions — this is the single best way to build speed and accuracy.
